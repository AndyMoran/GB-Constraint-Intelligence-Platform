# Part I — Data Platform & Signal Engineering (NB01–NB07)

## Scope

The Data Platform & Signal Engineering layer forms the foundation of the GB Constraint Analytics Platform.  
It constructs a clean, validated, and reusable analytical dataset describing GB electricity system behaviour.

This layer supports both the GB‑wide screening analysis and the detailed North‑West case study developed in Part II.

---

## Objectives

- Build a unified analytical dataset integrating constraint, BM, and market signals  
- Ensure timestamp alignment and data quality  
- Engineer derived metrics required for behavioural and economic analysis  
- Provide a stable, reusable foundation for future research

---

## Activities

- Data ingestion and validation  
- Timestamp harmonisation  
- Constraint dataset processing  
- Balancing Mechanism data preparation  
- Feature engineering  
- Derived signal construction  
- Master dataset assembly

---

## Key Outputs

- `market_master_signal_v2.parquet`  
- Constraint‑level analytical datasets  
- Derived loading and burden metrics  
- Common schemas used across Parts II and III
